Building a Recipe Blog with WordPress Playground

WordPress Playground lets you spin up a full WordPress instance right in your browser -- no server required. In this post, we will walk through building a recipe blog from scratch, complete with custom blocks and a theme tailored for food photography.

Getting Started

First, we need to set up our development environment. With Playground, this is as simple as opening a URL. The entire WordPress stack -- PHP runtime, database, and web server -- runs inside your browser using WebAssembly. Changes persist across sessions when you enable browser storage.

{
  "landingPage": "/wp-admin/post-new.php",
  "phpVersion": "8.3",
  "wp": "6.7",
  "steps": [
    {
      "step": "installPlugin",
      "pluginData": { "resource": "wordpress.org/plugins", "slug": "jetpack" }
    },
    {
      "step": "installTheme",
      "themeData": { "resource": "wordpress.org/themes", "slug": "flavor" }
    }
  ]
}

The Blueprint JSON above configures our Playground instance with PHP 8.3, WordPress 6.7, installs the Jetpack plugin for sharing features, and applies the Flavor theme designed for food blogs.

Creating the Recipe Block

Custom blocks are the heart of the WordPress editor. Our recipe block will include structured fields for ingredients, cooking time, difficulty level, and step-by-step instructions -- all rendered as a beautiful card in the editor and on the front end.

Block Attributes

  • title -- The recipe name displayed as the card heading
  • ingredients -- Array of ingredient objects with name, quantity, and unit
  • prepTime -- Preparation time in minutes (renders as human-readable)
  • cookTime -- Cooking duration, combined with prep for total time
  • servings -- Number of servings with adjustable scaling

WordPress Playground is not just a development tool -- it is a new way to think about WordPress distribution. Ship entire sites as URLs, share configurations as Blueprints, and let users try before they install.
